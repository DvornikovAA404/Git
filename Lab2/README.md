**Тема:** Разработка интерфейсных тестов для функционала Level 1 библиотеки OpenBLAS.

Описание задачи и окружения
----
Цель работы заключалась в создании набора тестов на C++ для проверки интерфейсов базовых линейно-алгебраических операций (CBLAS Level 1).

Реализация тестового покрытия
---
Написанные тесты покрывают следующие группы функций первого уровня для одинарной и двойной точности:
- Операции копирования и обмена: copy, swap.
- Масштабирование и обновление векторов: scal, axpy.
- Вычисление скалярных произведений: dot.
- Расчет норм и поиск индексов: nrm2, asum, iamax.
- Применение вращений: rot, rotm, rotg.

Поскольку интерфейс BLAS не имеет встроенных комплексных типов, передача таких массивов реализована через указатели void*. Массивы в памяти формировались с учетом чередования действительной и мнимой частей. Скалярные коэффициенты также передавались по указателю на массив из двух элементов.

Особенности обработки ошибок
---
В ходе выполнения работы была исследована механика обработки невалидных аргументов. Стандартный интерфейс CBLAS предоставляет функцию cblas_xerbla для перехвата ошибок входных данных.

Однако, тестирование показало, что функции Level 1 не вызывают cblas_xerbla. Поискав причину в интернете, нашел информацию о том, что проверка аргументов намеренно исключена разработчиками стандарта для минимизации накладных расходов процессора на ветвления. Передача некорректных аргументов в функции первого уровня приводит к неопределенному поведению или ошибке сегментации памяти.
По этой причине интерфейсные тесты реализованы исключительно для валидных входных параметров.

Результаты
---
Интерфейсные тесты успешно скомпилированы. При запуске с оригинальной библиотекой OpenBLAS все вызовы функций завершаются корректно. Обработка ошибок происходит только за счёт компилятора.
